# 키보드 접근성 테스트 가이드

## 🎹 1분 테스트 (마우스 치우고 시작!)

### 준비물
- ❌ 마우스는 사용하지 마세요!
- ✅ 키보드만 사용

---

## 📋 테스트 체크리스트

### 홈페이지 (/)

1. **페이지 로드 후**
   - `Tab` 키를 누르세요
   - 첫 번째로 "Skip to main content" 링크가 보이나요? ✅

2. **헤더 내비게이션**
   - `Tab` 키를 계속 누르세요
   - 순서대로 나타나나요?
     - Home → Projects → Services → Contact ✅
   - 각 링크에 파란색 테두리(focus indicator)가 보이나요? ✅
   - `Enter` 키로 링크를 클릭할 수 있나요? ✅

3. **Featured Projects 섹션**
   - `Tab` 키로 프로젝트 카드에 도달할 수 있나요? ✅
   - `Enter` 키로 프로젝트를 열 수 있나요? ✅

4. **Contact Form**
   - `/contact` 페이지로 이동
   - `Tab` 키로 모든 입력 필드를 순회할 수 있나요? ✅
     - Name → Email → Phone → Address → Project Type → Message → Submit
   - 각 필드에 포커스가 명확히 보이나요? ✅
   - `Enter` 또는 `Space`로 Submit 버튼을 누를 수 있나요? ✅

---

### 프로젝트 페이지 (/projects)

1. **필터 버튼**
   - `Tab` 키로 필터 버튼들에 도달
   - `Enter` 또는 `Space`로 필터 활성화 ✅
   - 선택된 필터가 시각적으로 구분되나요? ✅

2. **프로젝트 그리드**
   - `Tab` 키로 각 프로젝트에 도달 ✅
   - `Enter`로 프로젝트 상세 페이지 이동 ✅

---

### 프로젝트 상세 페이지 (/projects/[id])

1. **이미지 캐러셀**
   - 화면 중앙의 작은 동그라미(indicator)로 `Tab` 이동
   - `←` `→` (화살표 키)로 이미지 전환 ✅
   - `Home` 키로 첫 번째 이미지 ✅
   - `End` 키로 마지막 이미지 ✅

2. **Pause/Play 버튼**
   - `Tab` 키로 "Pause slideshow" 버튼 도달
   - `Enter`로 자동 전환 중지/시작 ✅

3. **썸네일 내비게이션**
   - `Tab` 키로 썸네일 이미지들 순회
   - `Enter`로 해당 이미지 선택 ✅

---

## ✅ 테스트 통과 기준

모든 항목에서:
- [ ] 마우스 없이 모든 기능 사용 가능
- [ ] 포커스가 어디에 있는지 항상 명확히 보임
- [ ] 논리적인 순서로 `Tab` 이동
- [ ] 포커스가 사라지거나 갇히지 않음

---

## 🎯 주요 키보드 단축키

| 키 | 기능 |
|---|---|
| `Tab` | 다음 요소로 이동 |
| `Shift + Tab` | 이전 요소로 이동 |
| `Enter` | 링크/버튼 클릭 |
| `Space` | 버튼 클릭, 체크박스 토글 |
| `Esc` | 모달/메뉴 닫기 |
| `←` `→` `↑` `↓` | 캐러셀, 슬라이더 조작 |
| `Home` | 첫 항목으로 |
| `End` | 마지막 항목으로 |

---

## 🐛 문제 발견 시

만약 키보드로 접근/조작이 안 되는 요소를 발견하면:

1. **어느 페이지인가?** (예: /projects)
2. **어떤 요소인가?** (예: 필터 버튼)
3. **무엇을 시도했나?** (예: Tab으로 도달 시도)
4. **결과는?** (예: 포커스가 건너뜀)

이 정보를 accessibility@hjsconstruction.com으로 보내주세요.

## 💡 알아두면 좋은 점

### 포커스 표시가 사라지는 경우
일부 브라우저/OS 조합에서 포커스가 명확하지 않을 수 있습니다.
이것은 정상이 아니므로 신고 바랍니다.

### 마우스 사용자에게는?
- 마우스 기능은 모두 그대로 작동합니다
- 키보드 접근성은 "추가" 기능입니다
- 일반 사용자는 차이를 느끼지 못합니다
